## 🔐 ไฟล์สำคัญที่แปลเป็นภาษาไทยแล้ว:

### 📁 ไฟล์คู่มือ:
- **`ETHICAL_TESTING_GUIDE.md`** - คู่มือการทดสอบอย่างมีจริยธรรม
- **`USAGE_GUIDE_TH.md`** - คู่มือการใช้งานภาษาไทย
- **`SESSION_GUIDE_TH.md`** - คู่มือการหา Instagram Session ID

### 🛠️ ไฟล์เครื่องมือ:
- **`session_template.json`** - แม่แบบ session (ภาษาไทย)
- **`validate_session.py`** - ตัวตรวจสอบ session (ภาษาไทย) 
- **`run_ethical_tests.sh`** - สคริปต์ทดสอบครบวงจร (ภาษาไทย)

## 🍪 วิธีหา Instagram Session ID ของคุณเอง:

### ขั้นตอนที่ 1: สร้างบัญชีทดสอบ
1. สร้างบัญชี Instagram ใหม่ด้วยอีเมลของคุณ
2. ใช้เฉพาะสำหรับการทดสอบ
3. **ห้ามใช้บัญชีหลักของคุณ**

### ขั้นตอนที่ 2: Login และหา Session ID
1. เปิด browser ไปที่ https://www.instagram.com
2. Login ด้วยบัญชีทดสอบของคุณ
3. กด **F12** เพื่อเปิด Developer Tools
4. ไปที่ **"Application"** → **"Storage"** → **"Cookies"** 
5. คลิก **"https://www.instagram.com"**
6. หา cookie ชื่อ **"sessionid"**
7. คัดลอกค่าใน column **"Value"**

### ขั้นตอนที่ 3: ตั้งค่า SugarGlitch
```bash
# คัดลอกแม่แบบ
cp session_template.json session.json

# แก้ไขไฟล์ session.json
nano session.json
```

**ใส่ session ID ของคุณ:**
```json
{
    "sessionid": "session-id-ที่คุณคัดลอกมา",
    "account_info": {
        "username": "ชื่อผู้ใช้ทดสอบของคุณ",
        "email": "อีเมลทดสอบของคุณ@gmail.com",
        "purpose": "การทดสอบความปลอดภัยอย่างมีจริยธรรม"
    }
}
```

## 🔧 การทำงานของระบบ:

### 1. Browser API (ลำดับแรก) 🥇
- ใช้ BrightData Browser API เข้าถึง Instagram
- ปลอดภัยและเสถียรที่สุด

### 2. Proxy Method (ลำดับสอง) 🥈  
- ใช้ BrightData proxy ถ้า Browser API ไม่ได้
- ผ่าน proxy server

### 3. Mock Data (ลำดับสุดท้าย) 🥉
- ใช้ข้อมูลตัวอย่างถ้าวิธีอื่นไม่ได้
- เพื่อทดสอบระบบ

## ⚠️ คำเตือนความปลอดภัย:

### สิ่งที่ต้องทำ ✅:
1. ใช้เฉพาะบัญชีทดสอบของคุณเอง
