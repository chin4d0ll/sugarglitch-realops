# 🔬 คู่มือการทดสอบ SugarGlitch อย่างถูกต้องตามจริยธรรม

## 📋 การตั้งค่าสภาพแวดล้อมทดสอบของคุณเอง

### 1. สร้างบัญชี Instagram สำหรับทดสอบ

สำหรับการวิจัยความปลอดภัยที่ถูกต้อง ให้สร้างบัญชีทดสอบของคุณเอง:

```
บัญชีทดสอบ 1: yourname_test1@gmail.com
บัญชีทดสอบ 2: yourname_test2@gmail.com
ชื่อผู้ใช้: your_test_account_1
ชื่อผู้ใช้: your_test_account_2
```

### 2. การได้ Session ID ของคุณ (วิธีถูกกฎหมาย)

#### วิธีที่ 1: Browser Developer Tools
1. เข้าสู่ระบบบัญชีทดสอบ Instagram ของคุณ
2. กด F12 เพื่อเปิด Developer Tools
3. ไปที่ "Application" → "Storage" → "Cookies"
4. หา `https://www.instagram.com`
5. คัดลอกค่า `sessionid`

#### วิธีที่ 2: Network Inspector
1. เข้าสู่ระบบบัญชีของคุณ
2. เปิด Developer Tools → แท็บ "Network"
3. รีเฟรชหน้าเว็บ
4. มองหา request ใดๆ ไปยัง Instagram
5. ใน Request Headers หา field `Cookie`
6. สกัดค่า `sessionid`

### 3. ตั้งค่า session.json

```json
{
    "sessionid": "session-id-จริงของคุณจากขั้นตอนที่-2"
}
```

### 4. รายการตรวจสอบการตั้งค่าบัญชีทดสอบ

- [ ] สร้างบัญชีทดสอบด้วยอีเมลของคุณเอง
- [ ] บัญชีเป็นเพื่อการทดสอบเท่านั้น
- [ ] คุณมีสิทธิ์อย่างชัดเจนในการทดสอบบัญชีเหล่านี้
- [ ] จะไม่เข้าถึงข้อมูลผู้ใช้จริง
- [ ] ผลลัพธ์จะใช้เพื่อการศึกษา/วิจัยเท่านั้น

### 5. การรันการทดสอบ

```bash
# ทดสอบด้วย session ของคุณ
python main.py

# ตรวจสอบการเชื่อมต่อ Browser API
node browser_api.js

# ทดสอบการเชื่อมต่อ proxy
python test_proxy.py
```

### 6. สิ่งที่ toolkit จะทดสอบ

✅ **การทดสอบที่ถูกต้อง:**
- วิธีการตรวจสอบ session
- เทคนิคการสกัดข้อมูล (ข้อมูลของคุณเอง)
- รูปแบบการโต้ตอบ API
- การวิจัยระเบียบวิธีความปลอดภัย

❌ **ไม่อนุญาต:**
- ทดสอบบัญชีที่คุณไม่ได้เป็นเจ้าของ
- เข้าถึงข้อมูลส่วนตัวของผู้ใช้คนอื่น
- การแฮ็ก session โดยไม่ได้รับอนุญาต
- การโจมตีในโลกแห่งความจริง

## 🛡️ แนวทางจริยธรรม

1. **ทดสอบเฉพาะบัญชีที่คุณเป็นเจ้าของและควบคุม**
2. **ใช้ข้อมูลจำลอง/ทดสอบสำหรับการสาธิต**
3. **เคารพข้อกำหนดการใช้บริการของ Instagram**
4. **ใช้เพื่อการศึกษา/วิจัยเท่านั้น**
5. **อย่าแชร์ session ID หรือข้อมูลรับรอง**
